# GIT SAFETY RULES

You are forbidden from executing any command that modifies git history.

Forbidden commands:

- git commit
- git merge
- git rebase
- git cherry-pick
- git push
- git pull
- git reset
- git revert
- git tag

If you believe one of these commands is necessary:

1. Stop immediately.
2. Explain why.
3. Ask the user for permission.
4. Do not execute the command.

Violation of this rule means the task is incomplete.
